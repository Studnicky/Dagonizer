- [ ] Cerebras: same query — expect a `gpt-oss-120b` response (toolUse: partial, may fall back to plain chat)

- [ ] Gemini API: paste key — expect a `gemini-2.0-flash` response
- [ ] Gemini Nano (Chrome desktop with flag): expect on-device response
- [ ] WebLLM (Chrome desktop with WebGPU): expect Phi-3.5 response after model download
- [ ] Ollama (with `OLLAMA_ORIGINS=http://localhost:5173 ollama serve`): expect local response
- [ ] Stub (mobile or explicit selection): expect seed-library-grounded canned response

## Known limitations (document, don't block)

- Real-API smoke is manual only — automated end-to-end tests against live providers require credentials that can't live in the workspace
- Cerebras `gpt-oss-120b` tool-call adherence is partial; the adapter's fallback path retries as plain chat when tools are signalled unsupported
- OpenRouter `:free` tier can route to providers that strip `tools`; capability is declared `'partial'` for this reason
- Gemini Nano uses `responseConstraint` for tool emulation — JSON-only path, no native function calling
- The example's `BaseLlmClient` is still domain-specific (Book / Candidate / Intent); pattern packages ship abstract bases that aren't yet used by the Archivist nodes (incremental refactor on roadmap)
